# 📋 노리(Nori) 제품 기획안 v2

> 0~7세 영유아 부모를 위한 **AI 즉석 맞춤 놀이 레시피** 앱.
> 본 문서는 전략·설계 합의안(v1)에 출시/운영 전수조사 결과를 반영한 개정판이다.

---

## 1. 한 줄 정의 (핵심가치)

> **"검색하지 말고, 지금 우리 집 상황을 말하면 — 우리 아이에 맞춰 AI가 오늘 할 놀이를 즉석에서 만들어준다."**

기능을 늘리지 않고 **이 하나에 올인**한다. 음성·커머스·영상·복잡한 프로필은 핵심 가치가 검증된 뒤 붙인다.

---

## 2. 타깃 & 차별화

- **1순위 페르소나:** "지금 아무것도 없고, 지쳤고, 애는 보채는" 부모 (페인 최강)
- **경쟁사 빈틈:** 차이의 놀이 등 기존 앱은 "나이(생년월일)"까지만 받고 **취향은 안 받는다** → 노리는 **나이 × 취향 × 상황** 조합으로 개인화
- **진부함 탈출 3축:**
  1. **조합(Combination):** 관심사 × 재료 × 상황을 버무려 "검색해도 안 나오는" 결과
  2. **기억(Memory):** 세션을 넘어 아이를 기억해 난이도·관심사를 이어줌 (락인 + 미래 ML 연료)
  3. **결정 제거(Activation):** "생각 말고 지금 이거 해" — 부모의 진짜 페인은 몰라서가 아니라 지쳐서

---

## 3. 핵심 플로우

```
0. 앱 켬 → 즉시 무료 놀이 1장 (입력 0, 로그인 0)          ← 가치 먼저(value-first)
       · "맨몸으로 바로 놀기" 디폴트
       · "집에 무조건 있는 물건"은 묻지 않고 AI 프롬프트 기본 전제로
1. (체험 후) 온보딩: 나이 1탭 → 나이맞춤 비주얼 취향 그리드(멀티탭)   ← 실질 2단계, <1분
2. 이후 모든 추천이 조합형 맞춤 레시피
3. 저장/찜하려는 순간 → 소셜 로그인 첫 등장 (게스트 모드 허용)
```

**설계 원칙**
- 진입 = 0클릭 즉시 결과. "입력→결과"가 아니라 "결과→(원하면)조정"으로 순서를 뒤집는다.
- 로그인은 "막는 문"이 아니라 "내 걸 저장하려고 스스로 여는 문".

---

## 4. 온보딩 설계

- **순서: 나이 먼저 → 그 나이에 가능한 취향 선택지만 노출** (과부하 방지 + 안전 + "똑똑해 보임")
- **모달리티: 비주얼 카드 그리드** (재인 > 회상). MVP는 이모지/심플 아이콘 → 검증 후 일러스트
- **2단계에서 끊는다.** 형제·성향 등은 가치 체험 후 점진 수집
- **"아이랑 같이 골라요"** — 온보딩을 교감 순간으로 (브랜드 톤)
- ⚠️ **발달 특이사항(건강=민감정보)은 MVP에서 수집하지 않는다** (§9 법무 참고)

## 5. 결과물(레시피) 포맷 — 고정

| 블록 | 내용 |
|---|---|
| 준비물 | 없음 / 집에 흔한 것 |
| 순서 | 3단계 |
| 한 줄 팁 | 실전 팁 1개 |
| 발달 포인트 | "이런 게 자라요" 1줄 |
| ⚠️ 안전 고지 | 보호자 감독 필수 + (위험 키워드 시) Red Flag 컴포넌트 |

---

## 6. 수익 모델 (레이어)

1. **교구·준비물 제휴 커머스** (맥락에 녹임) — 1순위, README 원기획과 일치
2. **프리미엄 구독** (무제한·맞춤 강화) — 2순위, 가치 체험 후
3. 절제된 네이티브 광고 — 보조만. **배너 도배 ❌** (신뢰 훼손)

- 비용: 레시피 1회 ≈ **1원 미만**(Gemini **Flash**) → **무료로 풀어도 안전**. 정밀 단가 튜닝은 사용자 증가 후.

## 7. 마케팅 앵글

- **"AI 놀이 추천"으로 팔지 않는다** (AI 단어는 피로감, 편리함은 약한 셀링).
- **리드 = 감정적 순간:** "비 오는 화요일 4시, 새 장난감도 없고 애는 보채고… 그때 켜세요."
- **증거 = 놀라운 조합:** "공룡 좋아하는 우리 애 + 양말 → 양말 공룡 인형극" (검색해도 안 나오는 결과를 스크린샷으로)
- **감성 포지셔닝:** "평균이랑 비교 안 해요. 오늘 우리 애가 웃으면 충분해요."
- **채널:** 맘카페·인스타 릴스·틱톡. **결과물 자체가 UGC** → 바이럴 효율 높음.
- ⚠️ 마케팅 매력의 9할은 **결과물 품질**에 묶임 → Phase 0(품질 실측)이 마케팅 성패까지 결정.

---

## 8. 기술 스택

- Next.js 15 / Supabase(Auth·DB·RLS) / **Gemini Flash**(대화·생성)
- 음성(Web Speech), 영상(YouTube API 링크), 추천 랭킹 ML → **전부 나중**
- **자체 딥러닝 모델 ❌** (Gemini가 이미 딥러닝). ML은 "이거 했어요" 데이터가 쌓인 뒤 추천 개인화에만 도입.
- "페르소나 커버"는 자체 모델이 아니라 **프롬프트 설계 + 입력 변수(나이·취향·상황)** 로 해결.

# 부록 B. TOP 3 블로커 해결 설계

## B1. 아동 민감정보 — "수집하지 않음"으로 회피

- **결정:** MVP는 **나이(개월/연령대 구간)** 와 **취향(비주얼 태그)** 만 수집. 발달특이사항·이름·생년월일 원본은 **받지 않는다.**
  - 나이도 정확한 생년월일 대신 **연령대 구간**(돌전/1-2세/3-4세/5세+)으로 받으면 개인식별성↓.
- **동의:** 온보딩 1회차에 **이용약관·개인정보처리방침 동의** 체크 1개. "수집 항목: 연령대, 취향 태그(아동 식별정보 아님)" 명시.
- **국외이전 고지:** 입력이 Gemini(구글)로 전송됨을 처리방침에 명시. 입력에 **이름 등 식별정보를 넣지 말 것**을 UI 가이드로 유도.
- **효과:** 민감정보·법정대리인 동의의 무거운 규제 자체를 **설계로 우회**. 핵심가치(조합 추천)는 그대로 성립.

## B2. 놀이 물리적 안전 — 3중 방어

1. **입력 단계:** 나이(연령대)를 항상 프롬프트에 포함.
2. **생성 단계(프롬프트 하드 제약):**
   ```
   - {연령대}에게 위험한 요소를 절대 제안하지 마라:
     · 영아(돌전): 작은 부품/삼킬 수 있는 것, 끈/비닐(질식), 물(익사), 높은 곳
     · 공통: 불/뜨거운 것, 날카로운 도구(가위·칼), 소형 자석/배터리, 알레르기 유발 음식
   - 모든 놀이는 보호자 감독 하에 가능한 수준이어야 한다.
   - 의학적 진단/치료/발달 평가를 하지 마라.
   ```
3. **출력 단계(2차 검증):** 생성 결과를 **금지 키워드/위험 패턴 필터**로 후처리. 걸리면 사용자에게 노출 전 **재생성 또는 안전한 fallback** 으로 대체.
4. **UI/면책:** 모든 레시피에 **"보호자 감독 필수"** 고지 + **"⚠️ 이 놀이가 위험해요" 신고 버튼**. 신고는 로그로 적재해 프롬프트/필터 개선에 환류.
5. **Red Flag 분리:** "발달 위험 신호"(예: "눈을 안 맞춰요")는 별도 하드코딩 컴포넌트로 **전문가 상담 권고**만 출력하고 AI 추측을 차단(README 원기획 유지).

## B3. 익명 호출 남용·비용 — rate limit + 식별 + 상한

- **식별:** 익명도 **Supabase anonymous sign-in** 으로 세션 발급(완전 무인증 호출 금지).
- **Rate limit:** 익명 세션·IP 기준 **분당/일일 생성 횟수 상한**(예: 익명 일 5회, 로그인 시 확대). 서버 액션/Route Handler 진입점에서 검사.
- **비용 가드:** Gemini 호출을 **단일 서버 모듈로 게이트** + **일일 호출 카운터** + 임계 초과 시 차단/큐잉. 클라우드 **예산 알림** 설정.
- **모델 고정:** 생성은 **Flash 고정**(Pro 금지)으로 단가 자릿수 유지.
- **남용 방어:** 임계 초과 시 캡차, 명백한 봇 패턴 차단.
